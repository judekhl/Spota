Spota AI Workflow



ChatGPT:

\- Strategy

\- Product decisions

\- Business model

\- App structure

\- Database planning

\- Debugging explanations

\- Writing prompts for Claude Code



Claude:

\- Code explanations

\- Breaking technical tasks into steps

\- Reviewing architecture



Claude Code:

\- Editing actual code

\- Running commands

\- Fixing bugs

\- Refactoring files



Cursor:

\- Opening and reviewing the codebase

\- Smaller code edits

\- Debugging exported Flutter code



FlutterFlow:

\- Building app screens visually

\- Connecting Supabase

\- Testing the app

\- Exporting code later



Supabase:

\- Database

\- Realtime parking availability

\- Operator/admin data



Rules:

\- Never ask AI to build the whole app at once.

\- Use one feature per prompt.

\- Test after every feature.

\- Save working versions.

\- Never paste secret API keys into public chats.

\- Do not let two AI agents edit the same files at the same time.
